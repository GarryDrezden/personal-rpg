# Промпт для генерации male-аватаров (стадии 1–20)

> **Текущий статус ассетов (male):** `stage-01.png`, `stage-02.png`, `stage-19.png`, `stage-20.png`, `death.png`. Стадии 3–18 пока отсутствуют — при просмотре подставляется ближайший якорь (1/2/19/20).

## Быстрый старт

1. Открой блок **«Готовый промпт»** ниже.
2. Замени **`{{STAGE}}`** на номер (1–20).
3. Замени **`{{STAGE_BODY}}`** на строку из таблицы **«Описание стадии»**.
4. Прикрепи референсы (см. **«Какие фото прикреплять»**).
5. Сгенерируй → сохрани как `public/game-assets/heroes/male/stage-XX.png` (две цифры).
6. Обрежь фон:

```powershell
python local/tools/process_hero_png.py public/game-assets/heroes/male/stage-01.png
```

Или все стадии в папке:

```powershell
python local/tools/process_hero_png.py public/game-assets/heroes/male
```

7. Увеличь `GAME_ASSET_VERSION` в `src/game/assetPaths.ts`.

## Какие фото прикреплять

| Стадии | Референсы |
|--------|-----------|
| **1–3** | **Фото A** (~180 кг) + при возможности `stage-20-approved.png` (стиль/лицо) |
| **4–6** | **Фото A** + **Фото B** (~161 кг) + `stage-20-approved.png` |
| **7–14** | `stage-20-approved.png` + ближайшая уже готовая стадия (5, 10…) |
| **15–19** | `stage-20-approved.png` + stage 15 или 10 |
| **20** | Утверждённый финал: `public/game-assets/heroes/male/_reference/stage-20-approved.png` |

**Пути референсов в проекте:**
- Фото A / B — храни в `public/game-assets/heroes/male/_reference/` (photo-a.png, photo-b.png) или прикрепляй из чата.
- Образец стиля и идентичности: `public/game-assets/heroes/male/_reference/stage-20-approved.png`

**Стратегия:** stage 20 — якорь стиля и лица. Ранние стадии строим от Photo A/B, поздние — интерполяция к stage 20, **не наоборот**.

```text
Используй приложенные фотографии как основные референсы внешности и телосложения персонажа.
Фото A — мужчина примерно в стартовой форме около 180 кг.
Фото B — тот же мужчина в более лёгкой форме около 161 кг.
Также используй приложенный stage-20-approved как образец стиля рендера, лица, одежды и идентичности (для стадий 1–19 — та же личность, но другое телосложение).

Нужно создать полноценный игровой аватар для personal RPG / gamified transformation app.

## Стадия
Стадия {{STAGE}} из 20.
Прогресс пути: см. таблицу стадий.
На этой стадии тело должно показывать:
{{STAGE_BODY}}

## Главная задача
Аватар явно похож на человека с референсных фото, но в стилистике приложения: премиальный stylized realistic 3D game art. Узнаваемая стилизованная версия этого мужчины, не generic model.

## Требования
- прозрачный фон (real alpha, без плашки, без комнаты, без ванной);
- персонаж в полный рост, от макушки до обуви;
- изолированный чистый силуэт для UI;
- без текста, логотипов, водяных знаков, фоновой сцены;
- стоит прямо, читается в уменьшенном размере.

## Стиль
- stylized realistic 3D game art, 30–40% stylization;
- премиальный mobile RPG;
- мягкий студийный свет;
- лёгкий dark fantasy light;
- не хоррор, не аниме, не мультяшность, не вектор, не гиперфото, не бодибилдер;
- благородная спокойная чуть суровая подача, без насмешки и гротеска.

## Внешность (сохранить на всех стадиях)
- взрослый высокий мужчина ~200 см, крупная широкая рама;
- светлая кожа, русая борода, волосы светло-/тёмно-русые зачёсаны назад;
- серьёзное собранное лицо, массивная шея, широкие плечи, крупные руки;
- татуировка на правом предплечье (стилизованно);
- оранжевые фитнес-часы на левой руке.

## Одежда (одинаковая на всех стадиях)
- тёмная однотонная футболка;
- чёрные спортивные шорты;
- тёмные кроссовки.
Посадка меняется: рано — натянута на животе; поздно — свободнее и собраннее.

## Поза
Фронтальная или лёгкий 3/4, уверенная устойчивая стойка, руки расслаблены, ноги на ширине плеч, по центру кадра.

## Логика трансформации
Постепенное снижение процента жира, не ранний набор мышц.
Ранние/средние стадии — всё ещё крупный мужчина.
Даже на 40–50% — не фитнес-модель.
Финал — здоровая лёгкая версия того же человека, не бодибилдер.

## Нельзя
фон, интерьер, бодибилдер, generic model face, резкое исчезновение живота, ранний пресс, карикатура, унизительная подача.

## Технически для экспорта
Solid black background ONLY for easy cutout to transparent PNG. No floor, no vignette card, no gray panel.
Full body large in frame, minimal empty margins.
```
